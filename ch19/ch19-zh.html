設計實驗

你需要多少資料？

收資料之前，兩個問題決定實驗的命運：

  • 要多少資料才能得到統計顯著的結果？
  • 給定你收得到的資料量，偵測得到多強的效果？

R 的 power.* 家族兩者都能回答。詞彙：顯著水準 ＝ 型一錯誤機率（偽陽性）；檢定力（power） ＝ 1 − 型二錯誤機率（1 − 偽陰性）。每個函式都會計算你留成 NULL 的那個參數。

實作範例：設計藥物試驗

測試抗憂鬱藥，以 HAMD 量表（0–48）度量憂鬱程度。假設標準差 8.9（出自 FDA 資料）、顯著水準 0.05、目標檢定力 0.95。每組 25 名受試者時，偵測得到多大的組間差異？

power.t.test(power=.95, sig.level=.05, sd=8.9, n=25)

     Two-sample t test power calculation 

              n = 25
          delta = 9.26214
             sd = 8.9
      sig.level = 0.05
          power = 0.95
    alternative = two.sided

NOTE: n is number in *each* group

至少 9.26 分的 HAMD 差距。受試者加倍：

power.t.test(power=.95, sig.level=.05, sd=8.9, n=50)

     Two-sample t test power calculation 

              n = 50
          delta = 6.480487
             sd = 8.9
      sig.level = 0.05
          power = 0.95
    alternative = two.sided

NOTE: n is number in *each* group

6.48 分就夠了。這就是實驗設計的實況：在花一毛錢收資料之前，先在樣本數與可偵測效果之間權衡。

t 檢定設計

power.t.test

power.t.test(n = NULL, delta = NULL, sd = 1, sig.level = 0.05,
             power = NULL,
             type = c("two.sample", "one.sample", "paired"),
             alternative = c("two.sided", "one.sided"),
             strict = FALSE)
引數 說明
n 每組觀測數
delta 真實的平均差
sd 真實標準差
sig.level 顯著水準（型一錯誤機率）
power 1 − 型二錯誤機率
type 雙樣本／單樣本／成對
alternative 雙尾或單尾
strict 雙尾時採嚴格詮釋

n、delta、sd、sig.level、power 五者給四個；留 NULL 的第五個就是被計算的對象。

比例檢定設計

power.prop.test

供以 prop.test 度量的實驗使用：

power.prop.test(n = NULL, p1 = NULL, p2 = NULL, sig.level = 0.05,
                power = NULL,
                alternative = c("two.sided", "one.sided"),
                strict = FALSE)

p1／p2 是兩組的成功機率；其餘同前。一樣：n、p1、p2、sig.level、power 給四個、算 NULL 的那個。

棒球的情境數據是不是鬼扯？

ESPN 愛閃「兩出局二壘有人時 10 打數 3 安打」這種數據。假設某打者在某情境打 .300、其他情境打 .260——要多少情境打數才能讓 .300 顯著（α ＝ 0.05、檢定力 0.95、單尾）？

power.prop.test(p1=.260, p2=.300, sig.level=0.05,
                power=.95, alternative="one.sided")

     Two-sample comparison of proportions power calculation 

              n = 2724.482
             p1 = 0.26
             p2 = 0.3
      sig.level = 0.05
          power = 0.95
    alternative = one.sided

NOTE: n is number in *each* group

超過 2,724 個打數——還是每組。反過來問：n ＝ 10 時能做到什麼？

power.prop.test(n=10, p1=.260, p2=.300, power=.95,
                sig.level=NULL, alternative="one.sided")$sig.level
[1] 0.9256439
power.prop.test(n=10, p1=.260, p2=.300, power=NULL,
                sig.level=.05, alternative="one.sided")$power
[1] 0.07393654

顯著水準 0.93，或檢定力 0.07。書中判決：多數情境數據是鬼扯——現在你能證明了。

ANOVA 檢定設計

power.anova.test

供以 ANOVA 分析的多組設計使用：

power.anova.test(groups = NULL, n = NULL,
                 between.var = NULL, within.var = NULL,
                 sig.level = 0.05, power = NULL)
引數 說明
groups 組數
n 每組觀測數
between.var／within.var 組間／組內變異數
sig.level／power 同前

這裡七個參數要給六個；留 NULL 的那個被計算。

power.anova.test(groups=4, between.var=1, within.var=3, power=.80)

     Balanced one-way analysis of variance power calculation 

         groups = 4
              n = 11.92613
    between.var = 1
     within.var = 3
      sig.level = 0.05
          power = 0.8

NOTE: n is number in each group

Tip

每個實證專案都該養成的習慣： 實驗之前先跑檢定力計算。它把一廂情願（「做完看看顯不顯著」）變成設計（「要偵測我們在乎的效果，每組需要 n 筆」）。
