擴充 R 的能力

什麼是套件？

套件（package）把相關的函式、說明文件與資料集打包成一個可安裝的單位——相當於 Perl 的 module、C 的 library 或 Java 的類別庫。

  • 有些套件隨 R 一起出貨（stats、graphics、utils 等）。
  • 公開軟體庫中還有數以萬計的套件，涵蓋金融、基因體學到最新的機器學習方法。
  • 你也可以自己打造套件。

Note

兩個分開的步驟。 套件必須先安裝到磁碟上的本地函式庫（一次性），再於每個要使用它的工作階段中載入。把兩者分開能節省記憶體、維持說明系統的速度，並避免套件之間的名稱衝突。

函式庫

函式庫（library）是磁碟上存放已安裝套件的目錄。

  • R 一定有一個系統層級的函式庫；其上可再疊加個人函式庫（沒有管理員權限時特別有用）。
  • .libPaths() 顯示 R 目前搜尋的函式庫。
.libPaths()
[1] "/Library/Frameworks/R.framework/Versions/4.6/Resources/library"

注意用語：你是把套件安裝進函式庫——儘管載入函式偏偏叫 library()。

列出套件

library()                    # 安裝了哪些？
installed.packages()[1:5, c("Package", "Version")]
# 目前載入了哪些？
(.packages())
[1] "stats"     "graphics"  "grDevices" "utils"     "datasets"  "methods"  
[7] "base"
  • installed.packages() 回傳一個矩陣，一列一個已安裝套件——名稱、版本、相依性、授權。
  • 新開的工作階段會自動載入一小組基礎套件。

載入套件

用 library() 載入已安裝的套件（或用 require()，套件不存在時回傳 FALSE 而非報錯）：

library(lattice)         # lattice 的函式現在可見了
"package:lattice" %in% search()
[1] TRUE
  • 載入會把套件掛上搜尋路徑（search path），即 R 解析名稱時查找的位置清單（第八章）。
  • detach("package:lattice") 可將其移除。
  • 各平台指令相同；GUI 只是提供同一個呼叫的選單捷徑。

CRAN

Comprehensive R Archive Network 是核心的公開軟體庫：

  • 超過兩萬個貢獻套件，各自附有文件與平台對應的編譯版。
  • 每個套件頁面列出相依性、手冊與長篇導覽（vignettes）。
  • Task Views（https://cran.r-project.org/web/views/）按領域整理套件——計量經濟、機器學習、時間序列、繪圖——是探索生態系的最佳入口。

你還會遇到其他軟體庫：Bioconductor（基因體學；第二十五章）與 GitHub（開發版）。

安裝套件

一個函式就能完成下載與相依性解析：

install.packages("ggplot2")                     # 單一套件
install.packages(c("dplyr", "lattice"))         # 一次多個
update.packages()                               # 全部更新
  • 在 RStudio：Tools → Install Packages… 不過是同一個呼叫包上對話框。
  • 每台機器安裝一次；每個工作階段 library() 一次。
  • 要從非 CRAN 軟體庫安裝可加 repos= 參數；GitHub 開發版則用 remotes 套件的 install_github()。

自製套件

當你累積了一批常用函式，打包成套件是自然的歸宿：

  • 套件就是一個目錄：DESCRIPTION 檔（中繼資料）、R/ 資料夾（程式碼）、man/（說明頁），可選的 data/。
  • package.skeleton() 能從工作空間的物件產生骨架；現代流程多用 devtools 與 roxygen2。
  • R CMD build 產生可散布的壓縮檔；R CMD check 執行完整的品質檢查。
R CMD build mypackage/
R CMD check mypackage_1.0.0.tar.gz

就算永遠不上架 CRAN，打包也是與隊友——或未來的自己——分享程式碼最乾淨的方式。
