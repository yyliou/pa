R 與生物資訊

Bioconductor 是什麼

全書大半與領域無關；本章是例外，聚焦生物資訊。Bioconductor 是一個開源的 R 套件專案，用於分析高通量基因體資料——微陣列、次世代定序等。它有自己的發布週期、自己的儲存庫（與 CRAN 分開），以及讓數百個套件互通的共用資料結構。

Note

安裝方式與 CRAN 不同。 Bioconductor 套件用自家安裝程式——以往是 biocLite，現在是 BiocManager：

install.packages("BiocManager")
BiocManager::install(c("affy", "GEOquery", "limma"))

實作範例

載入原始表現量資料

範例分析一份乳癌微陣列研究（GEO 編號 GSE2034，Affymetrix HG-U133A 平台）。Affymetrix 原始資料存在 .CEL 檔，用 affy 套件的 ReadAffy 批次讀入：

library(affy)
GSE2034 <- ReadAffy()                          # 讀工作目錄下所有 .CEL
# 或明確指名：
GSE2034.smpl <- ReadAffy(filenames=paste0("GSM", 36777:36800, ".CEL"))
GSE2034                                         # 一個 AffyBatch 物件

ReadAffy 回傳一個 AffyBatch——未處理的探針層級強度，外加把探針對應到 HG-U133A 上 22,283 個探針集的晶片定義（cdf）。

從原始資料到表現量

原始強度必須先品管、背景校正、正規化、依探針集彙總、取對數，才能分析：

library(simpleaffy)
qc(GSE2034)                          # 品管報告

# 一步完成正規化＋彙總（有多種方法）：
eset <- rma(GSE2034)                 # robust multi-array average -> ExpressionSet
# 或：vsnrma(GSE2034)、expresso(GSE2034, ...)

rma（及親戚 vsnrma、expresso）把 AffyBatch 收斂成一個 ExpressionSet——Bioconductor 的核心容器。

從 GEO 載入已處理資料

處理好的表現量集往往已公開——GEOquery::getGEO 直接下載：

library(GEOquery)
GSE2034.geo <- getGEO("GSE2034")     # 一個 ExpressionSet 串列
class(GSE2034.geo[[1]])
## [1] "ExpressionSet"

不需要 .CEL 檔——分析可以從單一下載開始。

ExpressionSet

核心資料結構的解剖

ExpressionSet 把一個實驗所需的一切打包，並附存取函式：

組件 存取函式 內容
表現量矩陣 exprs() 基因（列）× 樣本（欄）
表型資料 pData()／phenoData() 每樣本的詮釋資料（處置、結果……）
特徵資料 fData()／featureData() 每基因的註解
實驗資訊 experimentData() 研究層級描述
註解 annotation() 平台，例如 "GPL96" 
dim(exprs(eset))            # 基因 x 樣本
head(pData(eset))           # 樣本詮釋資料

對齊表型資料

真實研究需要把臨床／表型資料連結到表現量矩陣。替換或合併 pData：

# 建好表型資料再掛上
pData(eset) <- new("AnnotatedDataFrame", data=clinical.df)
# 或把 GEO 表型與自己的合併：
merged <- merge(pData(GSE2034.geo[[1]]), my.clinical, by="sample")
pData(GSE2034.geo[[1]]) <- merged

在兩個來源之間對齊樣本是麻煩所在——書中花了不少功夫讓列的順序對得上。

下游：差異表現

有了表型對齊的 ExpressionSet，標準的下一步是找出差異表現基因——套件 limma（微陣列線性模型）：

library(limma)
design <- model.matrix(~ pData(eset)$outcome)
fit <- lmFit(eset, design)           # 每個基因配一個線性模型
fit <- eBayes(fit)                   # 經驗貝氏調節
topTable(fit, number=20)             # 前 20 個差異表現基因

Tip

為什麼要獨立的生態系？ 基因體資料龐大、結構化（基因 × 樣本 × 註解）、方法繁多。Bioconductor 的共用 ExpressionSet（及後繼者如 SummarizedExperiment）讓正規化、檢定、註解、視覺化等數十個套件能彼此接合。你學的 R 直接通用；只有資料結構與安裝程式是特殊的。
