R 的格狀圖

歷史與理念

lattice 套件 (Sarkar 2008) 實作了 trellis 繪圖——Cleveland 為條件化資料設計的視覺化框架 (Cleveland 1993)：對資料的每個子集重複同一種面板，排成網格以便即時比較。

  • 公式介面：y ~ x | condition 讀作「y 對 x，condition 的每個水準一個面板」。
  • lattice 函式回傳物件；物件被列印時圖才出現——在腳本與函式中務必記得這點。
temp <- tempfile(fileext = ".rda")
download.file("https://raw.githubusercontent.com/cran/nutshell/master/data/births2006.smpl.rda", temp, mode = "wb")
load(temp)
dim(births2006.smpl)
[1] 427323     13

簡單範例

2006 年美國出生資料（10% 抽樣）：出生體重與生產方式有什麼關係？

histogram(~ DBWT | DMETH_REC, data = births2006.smpl,
          xlab = "Birth weight (g)")

一條公式產出三個對齊的面板——不用迴圈就完成條件化。

使用 lattice 函式

所有高階 lattice 函式共享同一套解剖構造：

xyplot(y ~ x | g,            # 公式，條件化可選
       data = df,            # 變數所在
       groups = h,           # 面板內的分組疊加
       layout = c(3, 1),     # 面板排列
       main = "...", xlab = "...", ylab = "...")
  • groups= 在面板內區分子集（配 auto.key=TRUE 出圖例）；| 在面板間區分子集。
  • 回傳的 "trellis" 物件可以儲存、用 update() 修改、之後再列印。

單變量圖：直方圖與密度圖

densityplot(~ DBWT | DPLURAL, data = births2006.smpl,
            plot.points = FALSE, layout = c(1, 5),
            xlab = "Birth weight (g)")

把面板直向堆疊（每種胎數一格：單胞胎、雙胞胎、三胞胎……）讓 x 軸對齊——多胞胎體重較輕，整個故事一欄講完。

單變量圖：點條與分位數

  • stripplot()——一筆觀測一個點，小群組的理想選擇；
  • qqmath()——樣本分位數對理論分布；
  • bwplot()——盒鬚圖，boxplot() 的 lattice 表親：
bwplot(DBWT ~ factor(APGAR5), data = births2006.smpl,
       xlab = "5-minute Apgar score", ylab = "Birth weight (g)")

體重與 Apgar 分數同升——二十個盒子一眼看盡。

雙變量圖

xyplot() 是條件化散佈圖。各星期幾的平均出生體重，依生產方式分面板：

wt <- aggregate(DBWT ~ DOB_WK + DMETH_REC, data = births2006.smpl, mean)
xyplot(DBWT ~ DOB_WK | DMETH_REC, data = wt, type = "h",
       layout = c(3, 1), xlab = "Day of week", ylab = "Mean weight (g)")

其他雙變量夥伴：barchart()、dotplot()（Cleveland 點圖——常是圓餅圖與長條圖的最佳替代品）。

三變量圖

三個數值維度時：

x <- seq(-2, 2, length.out = 40)
grid <- expand.grid(x = x, y = x)
grid$z <- with(grid, exp(-(x^2 + y^2)))
levelplot(z ~ x * y, data = grid)
  • levelplot()——彩色磚塊；contourplot()——帶標籤的等高線；
  • cloud()——3D 點雲；wireframe()——3D 曲面。

自訂面板函式

面板函式決定每個方格內畫什麼。包住預設函式再往上加：

xyplot(DBWT ~ WTGAIN | SEX,
       data = births2006.smpl[sample(nrow(births2006.smpl), 2000), ],
       alpha = 0.2,
       panel = function(x, y, ...) {
         panel.xyplot(x, y, ...)
         panel.lmline(x, y, col = "red", lwd = 2)
       },
       xlab = "Maternal weight gain (lb)", ylab = "Birth weight (g)")

panel.lmline、panel.loess、panel.abline、panel.grid 都是現成的積木。

自訂 lattice 外觀

  • trellis.par.get() / trellis.par.set() 全域調整主題（顏色、字型、符號）。
  • show.settings() 一眼展示目前主題。
  • 單張圖：par.settings= 把主題調整注入單一呼叫；
  • update(plot_object, ...) 修改既有的 trellis 物件——潤飾存好的圖而不必重算。
my.theme <- trellis.par.get()
my.theme$plot.symbol$pch <- 19
trellis.par.set(my.theme)

Lattice 與基礎繪圖的比較

基礎繪圖 Lattice
條件化 手動迴圈＋par(mfrow) 公式裡的 | g
回傳值 無（筆墨上紙） trellis 物件
圖例 手動 legend() auto.key = TRUE
面板間一致性 自己負責 自動

當問題是「這個模式在每個子群內都成立嗎？」時，lattice 正是利器——第十五章將介紹 ggplot2，第三套建立在完整文法上的系統。
