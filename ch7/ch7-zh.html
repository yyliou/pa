物件型錄

原始物件型別

每個 R 物件都有底層的型別，可用 typeof() 查看。整個型錄可分成幾個自然的家族：

  • 基本向量：logical、integer、double、complex、character、raw；
  • 複合物件：list 以及一切以串列為基礎的結構；
  • 語言物件：symbol、call、expression；
  • 函式：closure（R 層級）、builtin 與 special（以 C 實作）；
  • 特殊物件：NULL、環境，以及少數內部型別。
typeof(1.5); typeof(1L); typeof("a"); typeof(list()); typeof(mean)
[1] "double"
[1] "integer"
[1] "character"
[1] "list"
[1] "closure"

向量

向量是 R 的原子：有順序、同型別的集合。c() 會遞迴攤平它的引數：

v <- c(1, 2, c(3, 4, c(5)))
v
[1] 1 2 3 4 5
length(v)
[1] 5

名稱可以跟著元素走：

prices <- c(apple = 30, banana = 12, cherry = 95)
prices["banana"]
banana 
    12

串列

串列是異質容器，是 R 中一切複雜結構的骨幹：

rec <- list(name = "Kim", scores = c(88, 92), pass = TRUE)
rec$scores
[1] 88 92
rec[["name"]]
[1] "Kim"
rec[1]          # 單層中括號回傳（子）串列
$name
[1] "Kim"

Note

[ ] 與 [[ ]] 的差別。 單層中括號是切片，回傳串列；雙層中括號是取出單一元件，回傳元件本身。混淆兩者是最常見的串列錯誤。

矩陣與陣列

矩陣是帶著長度為 2 的 dim 屬性的向量；陣列推廣到任意維度：

m <- matrix(1:6, nrow = 2)
m
     [,1] [,2] [,3]
[1,]    1    3    5
[2,]    2    4    6
dim(m)
[1] 2 3
t(m)            # 轉置
     [,1] [,2]
[1,]    1    2
[2,]    3    4
[3,]    5    6
m %*% t(m)      # 矩陣乘法
     [,1] [,2]
[1,]   35   44
[2,]   44   56

列名與欄名讓矩陣能自我說明：dimnames(m) <- list(c("r1","r2"), c("a","b","c"))。

因子

因子（factor）儲存類別資料：內部是整數代碼，加上把代碼對應到標籤的 levels 屬性。

sizes <- factor(c("S", "L", "M", "L", "S"), levels = c("S", "M", "L"))
sizes
[1] S L M L S
Levels: S M L
table(sizes)
sizes
S M L 
2 1 2 
as.integer(sizes)     # 底層的代碼
[1] 1 3 2 3 1
  • ordered = TRUE 建立有序因子（S < M < L），比較運算子會尊重順序。
  • 因子是迴歸函式辨識「這欄是類別變數」的依據——虛擬變數會自動產生。

資料框

資料框——一列一筆觀測、一欄一個變數——技術上是一個等長向量的串列，類別為 data.frame：

df <- data.frame(player = c("A", "B", "C"),
                 yards  = c(43, 28, 51),
                 good   = c(TRUE, FALSE, TRUE))
str(df)
'data.frame':   3 obs. of  3 variables:
 $ player: chr  "A" "B" "C"
 $ yards : num  43 28 51
 $ good  : logi  TRUE FALSE TRUE
df[df$good, "yards"]
[1] 43 51

str() 是 R 裡最有用的檢視工具：結構、型別、預覽一次看完。

公式

公式不經求值地保存一段模型描述：

f <- yards ~ player + good
class(f)
[1] "formula"

公式語法：+ 加入項、- 移除項、* 交叉（a*b = a + b + a:b）、: 是交互作用、. 代表「其餘所有欄位」、I() 保護算術（I(x^2)）。模型、lattice 圖與彙總函式都接受公式。

時間序列

ts 物件替向量加上規律的時間中繼資料。依課程慣例，從 GitHub 載入課本的火雞價格序列：

temp <- tempfile(fileext = ".rda")
download.file("https://raw.githubusercontent.com/cran/nutshell/master/data/turkey.price.ts.rda", temp, mode = "wb")
load(temp)
class(turkey.price.ts)
[1] "ts"
start(turkey.price.ts); frequency(turkey.price.ts)
[1] 2001    1
[1] 12
plot(turkey.price.ts)

日期與時間

d <- as.Date("2026-09-22")
d + 7                          # 日期運算以天為單位
[1] "2026-09-29"
weekdays(d)
[1] "Tuesday"
now <- Sys.time()
class(now)                     # POSIXct：自 1970 起的秒數
[1] "POSIXct" "POSIXt" 
format(now, "%Y-%m-%d %H:%M")
[1] "2026-06-07 19:00"
  • 日資料用 Date；時間戳用 POSIXct。
  • difftime 物件表示時間長度；seq(d, by = "month", length.out = 3) 能產生日曆序列。

連線

連線（connection）把資料來源抽象化——檔案、URL、壓縮檔、管線——讓讀取函式不必在意位元組從哪裡來：

temp <- tempfile(fileext = ".rda")
download.file("https://raw.githubusercontent.com/cran/nutshell/master/data/dow30.rda",
              temp, mode = "wb")
load(temp)           # 我們的 GitHub 慣例：先下載、再載入

gz <- gzfile("big.csv.gz")    # 透明解壓縮
read.csv(gz)

課程慣例——先 download.file() 到暫存檔、再 load()——底層用的正是連線；mode = "wb" 確保二進位檔在各平台都完好無損。

屬性

屬性是讓普通向量升級成豐富結構的中繼資料：

x <- 1:6
attr(x, "dim") <- c(2, 3)     # 它變成矩陣了！
x
     [,1] [,2] [,3]
[1,]    1    3    5
[2,]    2    4    6
class(x)
[1] "matrix" "array" 
attributes(x)
$dim
[1] 2 3

names、dim、dimnames、levels、class 全都只是屬性——第十章的整套物件系統，就建立在這一個機制之上。
