名字住在哪裡

終於正式定義：環境

從第三章起我們一直繞著環境打轉；現在正式定義。環境（environment）是一個 R 物件，包含：

  • 某個脈絡下可用的符號集合、
  • 與這些符號關聯的物件（兩者合稱frame），以及
  • 指向父環境的指標。

每個求值脈絡都關聯一個環境。解析符號時，R 先搜尋目前的環境；找不到就遞迴往父環境搜尋，直到找到為止。

符號

解析發生在建構當下

定義變數，就是在某個環境中把符號指派給值。用符號組裝物件時，R 在建構當下就解析符號：

x <- 1; y <- 2; z <- 3
v <- c(x, y, z)
v
[1] 1 2 3
x <- 10      # v 早已建好；改 x 毫無影響
v
[1] 1 2 3

延遲求值：quote 與 eval

把運算式 quote 起來、之後再求值，就能延遲符號解析——每次 eval 都以當下的值解析：

x <- 1; y <- 2; z <- 3
v <- quote(c(x, y, z))
eval(v)
[1] 1 2 3
x <- 5
eval(v)
[1] 5 2 3

Promise：delayedAssign

promise 把求值延到變數第一次被使用時：

x <- 1; y <- 2; z <- 3
delayedAssign("v", c(x, y, z))
x <- 5
v          # 此刻才求值——用的是現在的 x
[1] 5 2 3

套件用 promise 讓物件「可見但不佔記憶體」。兩個遺憾：你無法檢測一個物件是不是 promise，也查不到它是在哪個環境建立的。

操作環境

環境工具箱

環境也是物件（內部以雜湊表儲存符號對應——第二十四章會利用這點寫出高效程式）。詞彙表：

函式 說明
assign／get／exists 在環境中繫結、取得、檢查名稱
objects 環境中定義的所有名稱
remove（rm） 從環境移除物件
search 已附加套件的名稱——即父環境的鏈
searchpaths 已附加套件的路徑
attach／detach 把串列、資料框或資料檔加入／移出搜尋路徑
emptyenv 空環境——所有鏈的終點
parent.env 某環境的父環境
baseenv base 套件的環境
globalenv／.GlobalEnv 使用者工作空間
environment 函式的環境（不給引數：目前環境）
new.env 全新的環境物件

objects 與 rm

兩個日常呼叫——這裡有什麼、把它清掉：

rm(list = ls())      # 先清空，從乾淨狀態開始
x <- 1; y <- 2; z <- 3
objects()
[1] "x" "y" "z"
rm(x)
objects()
[1] "y" "z"

全域環境

每個工作階段都有一個存放你物件的新環境：全域環境（global environment）。出乎意料的是，它不是環境樹的根——而是搜尋路徑鏈上的最後一個。從 .GlobalEnv 沿著父環境走：

e <- .GlobalEnv
while (environmentName(e) != environmentName(emptyenv())) {
  print(environmentName(parent.env(e))); e <- parent.env(e)
}
[1] "package:stats"
[1] "package:graphics"
[1] "package:grDevices"
[1] "package:utils"
[1] "package:datasets"
[1] "package:methods"
[1] "Autoloads"
[1] "base"
[1] "R_EmptyEnv"

已附加的套件一字排開成為父環境，最後到 base，終點是空環境——唯一沒有父環境的環境，所有鏈都通向它。

環境與函式

函式呼叫會建立環境

呼叫函式會建立新環境，引數在其中繫結成符號。（用語言學行話說：R 是詞法作用域（lexically scoped）的語言。）

env.demo <- function(a, b, c, d) { print(objects()) }
env.demo(1, "truck", c(1,2,3,4,5), pi)
[1] "a" "b" "c" "d"

只看得到引數——其他東西都住在父環境裡。並請牢記：函式的父環境是它被建立的地方，不是被呼叫的地方。在主控台定義的函式兩者恰好相同；來自套件的函式則不同。

呼叫堆疊

R 同時維護一個呼叫環境的堆疊（stack）（堆疊：只能從頂端加、從頂端拿——push 與 pop，像餐廳疊托盤）。每次呼叫推入一個環境；每次返回彈出一個。

函式 說明
sys.call 目前的函式呼叫（語言物件）
sys.frame 呼叫環境
sys.nframe 目前 frame 的編號（在主控台為 0）
sys.function 正在求值的函式
sys.parent 父 frame 的編號
sys.calls／sys.frames／sys.parents 整個堆疊的呼叫、環境、父 frame
sys.on.exit 目前 frame 的 on.exit 運算式
sys.status 前三者打包成一個串列
parent.frame sys.frame(sys.parent(n))——父 frame

為何在意？套件函式有時需要知道某符號在呼叫端的意義。建模函式正是如此：公式裡只有符號名、沒有資料，當你省略 data 引數時，模型函式就到呼叫環境裡找變數。

在別的環境求值：eval

eval 能在任意環境中評估運算式：

eval(expr, envir = parent.frame(),
     enclos = if (is.list(envir) || is.pairlist(envir))
                parent.frame() else baseenv())

書中範例：一個計時包裝函式——記下開始時間、在呼叫者的 frame 中評估引數、再印出耗時：

timethis <- function(...) {
  start.time <- Sys.time()
  eval(..., sys.frame(sys.parent(sys.parent())))
  end.time <- Sys.time()
  print(end.time - start.time)
}

計時示範

替一個逐格擴張向量的低效函式計時（主控台逐字稿）：

create.vector.of.ones <- function(n) {
  return.vector <- NA
  for (i in 1:n) {
    return.vector[i] <- 1
  }
  return.vector
}
timethis(returned.vector <- create.vector.of.ones(10000))
## Time difference of 1.485959 secs
length(returned.vector)
## [1] 10000

計時很有趣；但重點是最後一行——指派發生在呼叫環境裡，所以事後 returned.vector 確實存在。（順帶一提：迴圈前先用 length(return.vector) <- n 預先配置，時間從約 1.5 秒掉到約 0.04 秒——第二十四章的預告。）

簡寫：evalq、eval.parent、local

eval 的三個方便包裝：

  • evalq(expr, ...) ≡ eval(quote(expr), ...)——幫你 quote；
  • eval.parent(expr, n) ≡ eval(expr, parent.frame(n))——在呼叫者處求值；
  • local(expr) ≡ eval(quote(expr), envir=new.env())——在全新環境中求值。

用 eval.parent 能把計時函式寫得更短：

timethis.b <- function(...) {
  start.time <- Sys.time()
  eval.parent(...)
  end.time <- Sys.time()
  print(end.time - start.time)
}

把資料當環境用：with 與 within

把資料框或串列當成環境，就能像用符號一樣直呼其中的元素名：

example.list <- list(a=1, b=2, c=3)
a + b + c                          # 工作空間裡沒有……
Error:
! object 'a' not found
with(example.list, a + b + c)      # ……但 with() 找得到
[1] 6
within(example.list, d <- a + b + c)
$a
[1] 1

$b
[1] 2

$c
[1] 3

$d
[1] 6

with(data, expr) 評估並回傳結果；within(data, expr) 修改並回傳資料。

attach 與 detach

attach 把一個「資料庫」——串列、資料框或 save 存的資料檔——的物件加入搜尋路徑；detach 移除：

attach(what, pos = 2, name = deparse(substitute(what)),
       warn.conflicts = TRUE)
detach(name, pos = 2, unload = FALSE)

pos 是搜尋路徑位置，name 是附加資料庫的標籤（detach 時還會用到），warn.conflicts 控制名稱衝突警告，unload 控制是否卸載命名空間。

Warning

attach 請節制使用。 多個資料框欄名相同時，會真的分不清某個名字指的是誰的欄位。建議改用 transform 修改資料框欄位、用 with 對資料框評估運算式。

例外處理

錯誤與警告

不合法的運算式產生錯誤（error）；可疑的運算式產生警告（warning）：

12 / "hat"
Error in `12 / "hat"`:
! non-numeric argument to binary operator
if (c(TRUE, FALSE)) TRUE else FALSE
Error in `if (c(TRUE, FALSE)) ...`:
! the condition has length > 1

（第二個在書中年代只是警告；現代 R 已升格為錯誤。）為什麼在「環境」這章談例外？因為例外發生時，直譯器可能得放棄目前的函式、到呼叫環境去發出例外訊號——例外處理與環境密不可分。

發出訊號：stop、warning、message

告知使用者的三個層級，從致命到閒聊：

doWork <- function(filename) {
  if (file.exists(filename)) read.delim(filename)
  else stop("Could not open the file: ", filename)
}
doWork("file that doesn't exist")
Error in `doWork()`:
! Could not open the file: file that doesn't exist
doNoWork <- function(filename) {
  if (file.exists(filename)) "la la la"
  else warning("File does not exist: ", filename)
}
doNoWork("another file that doesn't exist")
Warning in doNoWork("another file that doesn't exist"): File does not exist:
another file that doesn't exist
doNothing <- function(x) {
  message("This function does nothing.")
}
doNothing("another input value")
This function does nothing.

寫自己的程式時：讓工作失效的錯誤用 stop；呼叫者該知道的問題用 warning；單純報訊息用 message。

攔截：try

假設 foo 呼叫 bar，bar 偶爾失敗（例如打不開檔案），而 foo 想要活下來改做別的事。簡單的工具是 try，它把大部分機制藏了起來：

res <- try({x <- 1}, silent=TRUE)
res
[1] 1
res <- try({open("file that doesn't exist")}, silent=TRUE)
res
[1] "Error in UseMethod(\"open\") : \n  no applicable method for 'open' applied to an object of class \"character\"\n"
attr(,"class")
[1] "try-error"
attr(,"condition")
<simpleError in UseMethod("open"): no applicable method for 'open' applied to an object of class "character">
class(res)
[1] "try-error"

try(expr, silent) 評估 expr；失敗時回傳 class 為 "try-error" 的物件（除非 silent=TRUE，否則順便印出錯誤訊息）。

攔截：tryCatch

更有力的工具是 tryCatch：一個運算式、一組處理器（handler）、加一個收尾運算式：

tryCatch(expression, handler1, handler2, ..., finally=finalexpr)

求值順序：先評估 expression；若出現條件（condition）（錯誤或警告），R 挑選引數類別與條件吻合的處理器；之後評估 finally——此時處理器已不再生效。小試一下：

tryCatch({
  log(-1)
}, warning = function(w) "caught a warning",
   finally = message("done either way"))
[1] "caught a warning"
