Constants

Numeric Vectors

Literal numbers default to double-precision floating point; suffixes and prefixes change the interpretation:

1.1                 # double
[1] 1.1
123L                # the L suffix forces an integer
[1] 123
1e3                 # scientific notation
[1] 1000
0xFF                # hexadecimal
[1] 255
1+2i                # complex
[1] 1+2i
  • class() distinguishes numeric (double), integer, and complex.
  • Very large or very small magnitudes print in scientific notation automatically.

Character Vectors

Strings may be quoted with double or single quotes; special characters are written with backslash escapes:

"hello"
[1] "hello"
'also a string'
[1] "also a string"
cat("tab:\tnewline:\nquote:\"done\"\n")
tab:    newline:
quote:"done"
  • \t, \n, \\, \" are the escapes you will actually use.
  • nchar() counts characters; paste() glues strings together.

Logical Constants and Symbols

TRUE; FALSE
[1] TRUE
[1] FALSE
  • Always spell out TRUE/FALSE. The shortcuts T/F are ordinary variables that someone can reassign — a classic trap.
  • A syntactic name starts with a letter (or a dot not followed by a digit) and contains letters, digits, dots, and underscores.
  • Any other name can still be used if wrapped in backquotes:
`my odd name` <- 42
`my odd name`
[1] 42

Operators

The Operator Zoo

7 %% 3              # modulus
[1] 1
7 %/% 3             # integer division
[1] 2
2 ^ 5               # exponentiation
[1] 32
1:3 %in% c(2, 3, 5) # set membership
[1] FALSE  TRUE  TRUE

Comparison (<, <=, ==, !=) and logical operators (!, &, |, &&, ||) return logical vectors:

c(1, 5, 10) > 4
[1] FALSE  TRUE  TRUE
TRUE & c(TRUE, FALSE)
[1]  TRUE FALSE
  • Single &/| are vectorized; double &&/|| examine one condition only and are reserved for if() tests.
  • Users can define new binary operators named %anything%.

Order of Operations

Operator precedence mostly matches school algebra — ^ before *// before +/-, arithmetic before comparison before logic, assignment last:

2 + 3 * 4 ^ 2
[1] 50
-2 ^ 2              # unary minus binds AFTER ^: this is -(2^2)
[1] -4
(-2) ^ 2
[1] 4

Tip

When in doubt, parenthesize. Parentheses cost nothing and document intent; see ?Syntax for the full precedence table.

Assignments

Several assignment forms exist:

x <- 5              # the standard
6 -> y              # rightward (rare)
z = 7               # allowed at top level
assign("w", 8)      # assignment as a function call
c(x, y, z, w)
[1] 5 6 7 8
  • Inside function calls, = names an argument, it does not assign — one more reason to standardize on <-.
  • The superassignment operator <<- modifies a variable in an enclosing environment (Chapter 8); use sparingly.
  • Replacement forms like names(x) <- ... and dim(x) <- ... look like magic but are ordinary function calls ("names<-").

Expressions

Separating and Grouping Expressions

a <- 1; b <- 2          # semicolons separate expressions on one line
{                       # braces group; value = last expression
  s <- a + b
  s * 10
}
[1] 30
  • Parentheses both override precedence and return their contents visibly:
(v <- c(1, 2, 3))       # assign AND print in one stroke
[1] 1 2 3

Control Structures

Conditional Statements

x <- 7
if (x > 5) "big" else "small"
[1] "big"
# vectorized version: one test per element
ifelse(c(1, 9, 3) > 5, "big", "small")
[1] "small" "big"   "small"

switch() selects among named alternatives:

grade <- function(letter)
  switch(letter, A = "excellent", B = "good", "see me")
grade("A"); grade("C")
[1] "excellent"
[1] "see me"

The unnamed final argument is the default case.

Loops: for, while, repeat

total <- 0
for (i in 1:5) total <- total + i
total
[1] 15
n <- 1
while (n < 100) n <- n * 2
n
[1] 128
  • repeat { } loops forever until break; next skips to the following iteration.
  • for iterates over any vector or list, not just numbers.

Note

Loops are often unnecessary. Vectorized arithmetic (x * 2) and the apply family (sapply, lapply, Chapter 9) express most iterations more clearly — and usually faster.

Loop Accumulators Done Right

Growing an object inside a loop re-allocates memory at every step. Pre-allocate instead:

squares <- numeric(10)       # pre-allocate
for (i in 1:10) squares[i] <- i^2
squares
 [1]   1   4   9  16  25  36  49  64  81 100
# the idiomatic one-liner
sapply(1:10, function(i) i^2)
 [1]   1   4   9  16  25  36  49  64  81 100

This habit matters enormously for performance (Chapter 24).
