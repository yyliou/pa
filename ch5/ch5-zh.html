R 的思考方式

文法課開始

學電腦語言很像學外語（只是簡單得多）。觀光客背幾句常用語就能應付；第三章給你的就是那些常用語。但想說得流利，就得學文法——動詞變化、句型結構。本章是 R 的文法課，由四個概念撐起：

  • 程式由運算式（expression）構成；
  • 運算式操作物件（object）；
  • 名稱是繫結到物件的符號（symbol）；
  • 工作由函式完成——而函式本身也是物件。

掌握這四點，第六章的細部語法就會顯得理所當然，而非武斷的規定。

運算式

R 程式是一連串的運算式：指派、條件判斷、算術等等——每個都會求值並產生結果：

x <- 1
if (1 > 2) "yes" else "no"
[1] "no"
127 %% 10
[1] 7

運算式由物件與函式組成，彼此之間可用換行或分號分隔：

"this expression will be printed"; 7 + 13; exp(0+1i*pi)
[1] "this expression will be printed"
[1] 20
[1] -1+1.224647e-16i

物件

所有 R 程式碼都在操作物件——電腦所表示的「東西」。數值向量、字元向量、串列、函式，全都是物件：

c(1, 2, 3, 4, 5)                       # 數值向量（五個元素）
[1] 1 2 3 4 5
"This is an object too"               # 字元向量（一個元素）
[1] "This is an object too"
list(c(1,2,3,4,5), "This is an object too", "this whole thing is a list")
[[1]]
[1] 1 2 3 4 5

[[2]]
[1] "This is an object too"

[[3]]
[1] "this whole thing is a list"
function(x, y) { x + y }               # 函式也是物件
function (x, y) 
{
    x + y
}

符號

正式地說，R 的變數名稱叫符號。把值指派給名稱，就是在目前的環境中把符號繫結到物件——而環境（有點套套邏輯地）定義為某個脈絡下已定義符號的集合：

x <- 1     # 在目前環境中，把符號 "x" 繫結到物件 1

第八章會完整討論符號與環境；現在只要記得：名稱和名稱所指的東西，是兩個不同的物件。

函式

所有工作都由函式完成

函式接收輸入物件（引數）、回傳輸出物件。每一個敘述——設變數、做算術、跑迴圈——都能寫成函式呼叫。書中的示範從一個樸素的向量開始：

animals <- c("cow", "chicken", "pig", "tuba")
animals[4] <- "duck"     # 修改第 4 個元素的正常寫法
animals
[1] "cow"     "chicken" "pig"     "duck"

那個中括號指派其實被剖析成對函式 [<- 的呼叫。下面是等價的寫法：

animals <- c("cow", "chicken", "pig", "tuba")
`[<-`(animals, 4, "duck")
[1] "cow"     "chicken" "pig"     "duck"

（一個細微差異：這樣寫結果不會自動印出——但效果完全相同。）

為什麼要知道「一切都是函式呼叫」？

實務上你絕不會寫 `[<-`(animals, 4, "duck")——中括號好讀太多了。但知道背後有個函式，意味著你能檢視它的程式碼、查它的說明頁，或為自己的類別寫同名方法（第十章）。更多「漂亮寫法 vs 函式寫法」的對照：

apples <- 3            # 漂亮的指派
`<-`(oranges, 4)       # 指派的函式寫法
apples + oranges       # 漂亮的算術
[1] 7
`+`(apples, oranges)   # 算術的函式寫法
[1] 7
if (apples > oranges) "apples are better" else "oranges are better"
[1] "oranges are better"
`if`(apples > oranges, "apples are better", "oranges are better")
[1] "oranges are better"
x <- c("apple","orange","banana","pear")
x[2]                   # 漂亮的向量取值
[1] "orange"
`[`(x, 2)              # 取值的函式寫法
[1] "orange"

指派時物件會被複製

在指派敘述中，多數物件是不可變的（immutable）：R 複製的是物件本身，不是物件的參照。（不可變是好事——能避免一整類錯誤，例如多執行緒程式裡的競態。）

u <- list(1)
v <- u
u[[1]] <- "hat"
u
[[1]]
[1] "hat"
v          # v 仍保有原值
[[1]]
[1] 1

向量、串列與多數基本物件都遵守這條規則。

函式呼叫也會複製

引數同樣會被複製。下面的函式把 x 的第 i 個元素設成 4——對外卻什麼也沒改變：

f <- function(x, i) { x[i] = 4 }
w <- c(10, 11, 12, 13)
f(w, 1)
w          # 原封不動：w 被複製給了 x
[1] 10 11 12 13

技術上，直譯器複製了繫結在 w 上的物件，再把符號 x 繫結到副本；修改只發生在函式的脈絡裡。（可變物件與傳參照，要靠環境才辦得到——第八章。）

Note

語意 vs 效能。 R 的行為彷彿每次指派都複製，但實際上常常就地修改——例如 v <- 1:100; v[50] <- 27 並不複製向量。語意相同，速度好得多。細節見 R Internals 手冊。

R 裡一切皆物件

不只資料：函式、符號、甚至運算式都是物件。函式名稱其實是指向函式物件的符號（這層關係又存放在環境物件裡）。同一個符號可以這一刻是數字、下一刻是函式：

x <- 1
x
[1] 1
x(2)                     # x（還）不是函式
Error in `x()`:
! could not find function "x"
x <- function(i) i^2
x
function (i) 
i^2
x(2)
[1] 4

你甚至能用 R 程式碼建構新函式——原則上，可以寫出會改寫自身定義的函式。

特殊值

NA：缺失值

NA（“not available”）代表缺失值——常見於載入的文字資料，也用來取代資料庫的 NULL。把結構撐大到原本沒定義的位置，新格子就填入 NA：

v <- c(1, 2, 3)
length(v) <- 4
v
[1]  1  2  3 NA

Inf、-Inf 與 NaN

大到無法表示的數會變成無限大，帶正負號——除以零也是同樣下場：

2 ^ 1024
[1] Inf
- 2 ^ 1024
[1] -Inf
1 / 0
[1] Inf

說不通的計算則得到 NaN（“not a number”）：

Inf - Inf
[1] NaN
0 / 0
[1] NaN

NULL

符號 NULL 永遠指向同一個特殊物件：空物件。

  • 常當函式引數使用，意思是「沒有提供值」；有些函式也會回傳它。
  • NULL 和 NA、Inf、-Inf、NaN 都不同——它根本不是一個值，而是值的缺席。
is.null(NULL)
[1] TRUE

型別轉換（Coercion）

自動轉型

用「不對的」型別呼叫函式，R 會嘗試轉型（coerce）讓呼叫照樣成功。兩種機制自動運作：

  • 正式物件： 泛型函式找不到完全吻合的方法時，會尋找能把物件轉成「有方法可用」型別的轉型方法（自訂方式見第十章）；
  • 內建型別： R 自動從較特定的型別轉向較一般的型別：
x <- c(1, 2, 3, 4, 5)
typeof(x); class(x)
[1] "double"
[1] "numeric"
x[2] <- "hat"          # 一個字元值就把整個向量拉了過去
x
[1] "1"   "hat" "3"   "4"   "5"  
typeof(x); class(x)
[1] "character"
[1] "character"

轉型規則

  • 邏輯值變成數字：TRUE → 1、FALSE → 0。
  • 值會轉成足以表示全部資訊的最簡型別。
  • 順序大致是：logical < integer < numeric < complex < character < list。
  • raw 型別不會被轉成其他型別。
  • 轉型時物件的屬性會被丟棄。
  • 想在傳引數時抑制轉型，用 I()（即 AsIs 函式）包起來——見 ?AsIs。

Note

為什麼這麼寬鬆？ 強型別語言（如 Java）遇到型別不符會丟例外、不會代為轉換。S 語言的設計者 John Chambers 回憶：早期 S「盡量讓各種情況都能動」，但後期較正式的結論是——不分情況自動把豐富型別轉成簡單型別，會產生令人困惑、因而不可信的結果。日常實務中——同型別的數值向量——轉型很少咬人。

R 直譯器

剖析、代換、求值

R 是直譯式語言：直譯器直接執行你的運算式，沒有編譯步驟（不像 C、C++、Java；像 LISP、Perl、JavaScript）。對每個運算式，直譯器會：

  1. 剖析（parse）：把語法糖轉譯成函式形式；
  2. 代換（substitute）：在適當處以物件代換符號；
  3. 求值（evaluate）：回傳一個物件——複雜運算式則遞迴進行。

例：輸入 x <- 1 會變成呼叫 `<-`(x, 1)——在目前環境把 1 繫結給 x，並回傳 1。

逐步看一個條件式

設 x 繫結為 1，考慮：

x <- 1
if (x > 1) "orange" else "apple"
[1] "apple"

直譯器把整行剖析成 if-then-else 結構，先評估條件 (x > 1)——為假——因此從未評估 "orange"；它評估 else 之後的運算式並回傳 "apple"。在這類特殊情況下，不是所有引數都會被評估——這正是 if 不能是普通函式的原因。而主控台上運算式回傳的東西，一律交給 print 處理。

.Internal 與 .Primitive

有些功能實作在 R 系統內部：

  • 許多函式透過 .Internal 呼叫內部系統程式碼——例如繪圖主力：
plot.xy
function (xy, type, pch = par("pch"), lty = par("lty"), col = par("col"), 
    bg = NA, cex = 1, lwd = par("lwd"), ...) 
{
    if (is.null(type)) 
        type <- "p"
    type <- as.character(type)
    if (length(type) != 1L || !nzchar(type) || is.na(type)) 
        stop(gettextf("invalid plot type"))
    if (nchar(type) > 1L) 
        warning(gettextf("plot type '%s' will be truncated to first character", 
            type))
    t <- substr(type, 1L, 1L)
    if (!isTRUE(t %in% c("l", "o", "b", "c", "s", "S", "h", "p", 
        "n"))) 
        stop(gettextf("invalid plot type '%s'", t))
    invisible(.External.graphics(C_plotXY, xy, t, pch, lty, col, 
        bg, cex, lwd, ...))
}
<bytecode: 0xc801a2200>
<environment: namespace:graphics>
  • 連 .Internal 的呼叫成本都嫌高的地方，函式就完全以 .Primitive 內部實作——只保留給少數效能攸關的基本功能：
`<-`
.Primitive("<-")

觀察 R 如何運作

quote：只剖析、不求值

因為 R 運算式就是 R 物件，你可以剖析（或部分求值）運算式並檢視結果。quote() 函式剖析引數但不求值，回傳一個語言物件（language object）：

typeof(quote(if (x > 1) "orange" else "apple"))
[1] "language"
quote(if (x > 1) "orange" else "apple")   # 語言物件的 print 方法不太有料……
if (x > 1) "orange" else "apple"

剖析樹

……但把語言物件轉成串列，結構就現形了——這是運算式的剖析樹：

as(quote(if (x > 1) "orange" else "apple"), "list")
[[1]]
`if`

[[2]]
x > 1

[[3]]
[1] "orange"

[[4]]
[1] "apple"

對每個元素套用 typeof，替整棵樹貼標籤（這裡用 lapply——而且拜轉型之賜，as 其實可以省略）：

lapply(as(quote(if (x > 1) "orange" else "apple"), "list"), typeof)
[[1]]
[1] "symbol"

[[2]]
[1] "language"

[[3]]
[1] "character"

[[4]]
[1] "character"

觀察：else 關鍵字在剖析中消失了；第一個元素是指向 if 函式的符號——所以連特殊語法都化成函式名在前、引數在後，跟任何呼叫一模一樣。常數的話，串列只有一項：as.list(quote(1))。

語法糖現形，以及 deparse

中括號查值 x[2] 和它的函式雙胞胎，剖析出完全相同的樹：

as.list(quote(x[2]))
[[1]]
`[`

[[2]]
x

[[3]]
[1] 2
as.list(quote(`[`(x, 2)))
[[1]]
`[`

[[2]]
x

[[3]]
[1] 2

這個轉譯不可逆——兩者塌縮成同一棵樹——但 deparse 能把剖析樹轉回排版正確的 R 程式碼，而且選的是漂亮寫法：

deparse(quote(x[2]))
[1] "x[2]"
deparse(quote(`[`(x, 2)))
[1] "x[2]"

Tip

讀完全書都帶著的工具： quote、substitute、typeof、class、methods 讓你親眼看直譯器怎麼剖析與分派。R 的行為讓你吃驚時，它們就是你的顯微鏡。
