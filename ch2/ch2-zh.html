與 R 面對面

一套系統的多種面貌

當有人說自己「在用 R」時，他們實際的操作方式可能大不相同。本章將逐一介紹主要的幾種：

  • Windows 與 macOS 上隨 R 附帶的圖形使用者介面（GUI）；
  • R 主控台（console）——所有介面核心的互動式提示字元；
  • 批次模式（batch mode）——不需互動、直接執行腳本；
  • RStudio——本課程使用的整合開發環境（IDE）；
  • 其他你日後可能遇到的前端工具。

無論外殼換成哪一種，底層的引擎——直譯器——完全相同。

R 的圖形使用者介面

Windows

  • Windows 版的 R 會安裝一個叫 RGui 的程式，開啟後是一個含選單列、工具列，以及最重要的 R 主控台的視窗。
  • 選單提供載入／儲存工作空間、安裝套件、簡單的腳本編輯等便利功能。
  • 這個 GUI 堪用但陽春，幾乎所有人很快就會改用 RStudio。

macOS

  • macOS 安裝程式提供原生應用程式 R.app：主控台視窗、具語法上色的簡易編輯器，以及管理套件與繪圖裝置的選單。
  • 圖形會顯示在獨立的 Quartz 繪圖視窗中。
  • 與 Windows 相同，內建 GUI 只涵蓋基本需求，正式工作多半在 IDE 中進行。

Linux 與 Unix

  • 多數 Linux 系統根本沒有附 GUI：在終端機輸入 R，主控台就直接在命令列啟動。
  • 圖形開在 X11 視窗中，其餘一切都在提示字元下完成。
  • 這是完全完整的 R 使用方式——GUI 元素只是便利品，不是必需品。

R 主控台

互動式提示字元

主控台是 R 真正發生的地方。在 > 提示字元後輸入運算式，R 會求值並印出結果：

17 %% 5
[1] 2
"hello, world"
[1] "hello, world"
  • 若一行語法尚未完整，提示字元會變成 +，R 等你把剩下的部分打完。
  • 沒有指派給變數的結果會直接印出後丟棄。

Note

讀取–求值–輸出迴圈（REPL）。 主控台不斷重複三個步驟：讀取你輸入的內容、求值、印出結果。像 R 這樣的互動式語言讓探索的成本極低——你可以一行一行地檢視資料。

命令列編輯

主控台內建的編輯功能可以省下大量打字：

  • 上下方向鍵捲動指令歷史；叫回先前的指令、修改後重新送出。
  • Tab 自動補全會接完函式與變數名稱——輸入 summ 後按 Tab 試試。
  • history() 顯示最近輸入的指令。
  • 各平台皆可用 Ctrl+C（或 Esc）中斷執行中的計算，回到提示字元。

批次模式

對於重複性的工作——每日報表、模擬、批改腳本——你根本不需要互動式工作階段。R 可以直接從作業系統的 shell 執行腳本：

# 傳統介面
R CMD BATCH myscript.R output.txt

# 現代且更有彈性的介面
Rscript myscript.R arg1 arg2
  • Rscript 將結果印到標準輸出並接受命令列引數，是自動化與資料管線的自然選擇。
  • 互動式環境能做的事，腳本全都能做。

在其他應用程式中使用 R

  • 透過第三方增益集，R 可以嵌入 Microsoft Excel，讓試算表使用者對工作表範圍呼叫 R 函式。
  • 資料庫系統、網頁伺服器與其他語言（Python、Java）都有連接 R 的橋樑。
  • 常見的模式是：其他程式準備資料，R 負責統計，結果再流回去。

RStudio

本課程使用的 IDE

RStudio Desktop 是免費、開源的 IDE，已是 R 事實上的標準前端。單一視窗整理成四個窗格：

  • 原始碼編輯器——腳本與文件，含自動補全與語法上色；
  • 主控台——與往常相同的 R 提示字元；
  • Environment / History——工作空間中的物件；
  • Files / Plots / Packages / Help——其餘一切。
# RStudio 是 R 的介面，不是 R 的替代品：
# 先安裝 R（第一章），再從
# https://posit.co/download/rstudio-desktop 安裝 RStudio。

為什麼 IDE 很重要

  • 一個快捷鍵（Ctrl/Cmd+Enter）就能執行目前的程式行或選取範圍，讓你在編輯器與主控台之間保持流暢。
  • 整合的說明文件、圖形與套件管理，免去不斷切換視窗的麻煩。
  • 「專案」（Projects）讓每個分析擁有自己的資料夾與工作空間——這個習慣會讓你的課堂作業井然有序。
  • RStudio 也能一鍵編譯本課程使用的文件格式（R Markdown、Quarto）。

其他執行 R 的方式

你也可能遇到：

  • ESS（Emacs Speaks Statistics）——在 Emacs 編輯器中使用 R，深受資深使用者喜愛。
  • Jupyter notebooks——資料科學圈流行的筆記本介面，可掛載 R 核心。
  • Visual Studio Code——擴充功能提供 R 編輯與整合終端機。

它們驅動的都是同一個直譯器；哪個讓你有生產力就用哪個。本課程的示範一律使用 RStudio。
