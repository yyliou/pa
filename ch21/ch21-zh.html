預測類別

迴歸的類別表親

分類模型預測類別反應——垃圾信或正常信、患病或健康、屬於哪一類。工具與第二十章相互呼應，但反應變數是因子。本章的貫穿範例是 Spambase：4,601 封電子郵件、57 個數值特徵，各標記為垃圾信或否。

我們從線性方法（邏輯斯迴歸、判別分析）走到樹與集成（bagging、boosting、隨機森林），再到神經網路與 SVM。

線性分類模型

邏輯斯迴歸

二元反應的主力：family=binomial 的 glm，建模結果的對數勝算（log-odds）：

spam.model <- glm(is_spam ~ ., data=spambase, family=binomial(link="logit"))
summary(spam.model)
predict(spam.model, newdata, type="response")   # 預測機率
  • summary 報告係數、z 值、p 值，外加虛無與殘差離差（deviance）及 AIC（不是 R²）。
  • type="response" 回傳機率；設門檻（如 0.5）轉成類別標籤。
  • 「Number of Fisher Scoring iterations」報告 IRLS 配適多快收斂。
  • 多類別延伸：nnet::multinom（多項對數線性模型），或有序結果用 MASS::polr。

線性與彈性判別分析

判別分析找出最能分隔類別的（線性）預測變數組合——MASS::lda（二次邊界用 qda）：

library(MASS)
iris.lda <- lda(Species ~ ., data=iris)
predict(iris.lda, iris)$class          # 預測類別

更豐富的變體在套件 mda：

  • fda——彈性判別分析（透過迴歸做非線性邊界）；
  • mda——混合判別分析（每類別為高斯混合）。

LDA 假設各類別共變異數相等、預測變數大致常態；QDA 鬆綁共變異數相等的假設。

對數線性模型

對列聯表中的計數，對數線性模型檢驗類別變數間的關聯——stats::loglin(table, margin, ...)（更友善的公式介面是 MASS::loglm）：

library(MASS)
loglm(~ Sex + Class + Survived, data=Titanic)   # 互相獨立
loglm(~ Sex*Class + Survived, data=Titanic)     # Sex、Class 有關聯

loglin 的引數：表格、要配適的邊際向量清單、可選的起始估計、maxit（IPF 演算法的迭代上限）。

樹與集成

分類樹：rpart

rpart 以遞迴分割建樹——反覆挑出最能分隔類別的特徵來切：

library(rpart)
spam.tree <- rpart(is_spam ~ ., data=spambase)
spam.tree                                  # 各分裂的文字描述
printcp(spam.tree)                         # CP 表：複雜度 vs. 誤差
plot(spam.tree); text(spam.tree)           # 畫出來
spam.tree.pruned <- prune(spam.tree, cp=0.01)   # 修剪避免過度配適

printcp／plotcp 的輸出（CP、nsplit、rel error、xerror）指引修剪：選使交叉驗證誤差最小的複雜度參數。

Bagging 與 Boosting

單棵樹不穩定；樹的集成準得多：

  • Bagging（拔靴聚合）——ipred::bagging 或 adabag::bagging：對拔靴樣本配多棵樹、平均／投票。降低變異。
  • Boosting——ada::ada 或 adabag::boosting（AdaBoost）、gbm::gbm（梯度提升）：依序配樹，每棵聚焦前一棵的錯誤。同時降低偏誤與變異。
library(adabag)
spam.bag  <- bagging(is_spam ~ ., data=spambase)
spam.boost <- boosting(is_spam ~ ., data=spambase)
predict(spam.boost, newdata)$class

這些是極易平行的——每棵樹獨立配適（第二十六章）。

隨機森林

randomForest 既對樹做 bagging、又在每個分裂隨機限制可考慮的特徵，使樹彼此去相關——通常是最強的現成分類器：

library(randomForest)
spam.rf <- randomForest(is_spam ~ ., data=spambase, ntree=500)
spam.rf                                # OOB 誤差估計＋混淆矩陣
importance(spam.rf)                    # 變數重要性
predict(spam.rf, newdata)

關鍵旋鈕：ntree（樹數）、mtry（每次分裂試的特徵數）。袋外（OOB）誤差是免費又誠實的準確度估計——不必另留測試集。

神經網路與 SVM

神經網路

nnet::nnet 配適單一隱藏層的前饋網路：

library(nnet)
spam.nn <- nnet(is_spam ~ ., data=spambase, size=10, decay=0.01, maxit=200)
predict(spam.nn, newdata, type="class")

size 設隱藏單元數、decay 是權重正則化（抑制過度配適）、maxit 限制迭代。（現代深度學習住在 keras／torch，想法相同、規模放大。）

支援向量機

e1071::svm 找最大邊際的邊界，以核函數處理非線性：

library(e1071)
spam.svm <- svm(is_spam ~ ., data=spambase,
                type="C-classification", kernel="radial")
predict(spam.svm, newdata)
naiveBayes(is_spam ~ ., data=spambase)   # 同套件還有單純貝氏

type="C-classification" 是標準分類器；kernel 可為 "linear"、"polynomial"、"radial"（RBF）、"sigmoid"。用 tune 調 cost 與核參數。同套件的 naiveBayes 提供快速的機率式基準。

k 最近鄰，以及如何選模型

最簡單的分類器——class::knn(train, test, cl, k)——以最近 k 個鄰居的多數決替每點標記；根本沒有「配適」模型。

Tip

沒有萬用最佳解。 邏輯斯迴歸好解釋；LDA 要檢查的假設少；樹好讀；隨機森林與 boosting 通常在純準確度上勝出；SVM 在高維出色；kNN 是快速基準。用獨立測試集或交叉驗證誠實比較，並以混淆矩陣判斷、別只看整體準確度。
