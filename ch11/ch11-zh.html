資料進，資料出

在 R 內輸入資料

小型資料集用第二部分學過的建構函式就夠了：

salary <- c(18700000, 14626720, 14137500)
position <- c("QB", "QB", "DE")
top.contracts <- data.frame(position, salary)
top.contracts
  position   salary
1       QB 18700000
2       QB 14626720
3       DE 14137500
  • rbind() 增加列；cbind() 增加欄。
  • scan() 可互動式逐筆輸入——快速實驗很方便。

編輯 GUI

R 內建一個陽春的試算表式編輯器：

top.contracts <- edit(top.contracts)  # edit 回傳「新的」物件
fix(top.contracts)                    # fix 會自己把結果存回去

Warning

edit() 不會就地修改。 忘記把它的結果指派回去是經典的資料遺失時刻；fix() 的存在正是為了避免這件事。

  • 編輯器外觀依平台而異（Windows 表格、macOS 表單、X11 視窗），但函式完全相同。
  • 真正的資料輸入用試算表軟體更舒服——編輯器最適合快速檢視與修一兩個儲存格。

儲存物件

save() 把任意一組 R 物件寫成可攜的二進位檔；load() 連名稱一起還原：

tmp <- tempfile(fileext = ".RData")
save(top.contracts, file = tmp)       # `file =` 必須具名
rm(top.contracts)
load(tmp)
top.contracts[1, ]
  position   salary
1       QB 18700000
  • 檔案跨平台通用；預設開啟壓縮。
  • save.image() 快照整個工作空間（離開時的「Save workspace?」就是它）。
  • saveRDS()/readRDS() 儲存單一物件且不含名稱——程式化使用往往更乾淨。

從網路載入：課程慣例

download.file() 把任何 URL 上的檔案抓到磁碟（二進位檔要用 mode = "wb"）；接著用 load() 還原。本課程所有資料集正是這樣載入的：

temp <- tempfile(fileext = ".rda")
download.file("https://raw.githubusercontent.com/cran/nutshell/master/data/top.bacon.searching.cities.rda", temp, mode = "wb")
load(temp)

匯入文字檔

read.table() 家族把分隔文字剖析成資料框：

df  <- read.table("file.txt", header = TRUE, sep = "\t")
csv <- read.csv("file.csv")             # 逗號分隔
eu  <- read.csv2("file.csv")            # ; 分隔、, 小數點
fwf <- read.fwf("file.dat", widths = c(8, 4, 4))

值得背起來的引數：header、sep、na.strings、stringsAsFactors、skip、colClasses（大檔案的速度利器）。

# readLines 顯示原始文字行——剖析前一定先看檔案長相
readLines(textConnection("a,b\n1,2\n3,4"))
[1] "a,b" "1,2" "3,4"

其他軟體與試算表

  • foreign 套件可讀傳統統計軟體格式：SAS（read.xport）、SPSS（read.spss）、Stata（read.dta）等。
  • Excel 檔：readxl 套件（read_excel("book.xlsx")）不需安裝 Excel；openxlsx 還能寫出。
  • 土法煉鋼依然有效：先把工作表匯出成 CSV，再 read.csv()。
library(readxl)
roster <- read_excel("stu/roster.xlsx", sheet = 1)

匯出資料

每個匯入函式都有鏡像：

write.csv(top.contracts, "contracts.csv", row.names = FALSE)
write.table(top.contracts, "contracts.txt", sep = "\t")
saveRDS(top.contracts, "contracts.rds")
  • row.names = FALSE 可避免那個讓試算表使用者一頭霧水的多餘首欄。
  • 與其他語言交換資料，CSV 仍是通用語；R 對 R，請用 .rds。

從資料庫匯入

三種策略，整合度遞增：

  1. 先匯出再匯入：在資料庫工具中把查詢結果倒成 CSV，再 read.csv()。粗糙但可靠。
  2. RODBC：透過 ODBC 驅動程式連線——幾乎所有有驅動程式的資料庫都行。
  3. DBI：R 原生的資料庫介面，依引擎搭配後端套件（RSQLite、RPostgres、RMariaDB……）。

RODBC 速覽

library(RODBC)
ch <- odbcConnect("dsn_name", uid = "user", pwd = "pass")
df <- sqlQuery(ch, "SELECT player, yards FROM kicks WHERE good = 1")
sqlTables(ch)              # 列出資料表
close(ch)
  • sqlFetch(ch, "tbl") 抓整張表；sqlSave() 把資料框寫回去。
  • ODBC 的設定發生在 R 之外（驅動程式＋資料來源名稱），實務上那才是主要工作量。

DBI 速覽

library(DBI)
con <- dbConnect(RSQLite::SQLite(), ":memory:")   # 用完即丟的資料庫
dbWriteTable(con, "contracts", top.contracts)
dbGetQuery(con, "SELECT position, MAX(salary) AS top FROM contracts GROUP BY position")
  position      top
1       DE 14137500
2       QB 18700000
dbDisconnect(con)
  • 一套泛用 API——dbConnect、dbGetQuery、dbWriteTable、dbDisconnect——背後是可互換的引擎。
  • 預備敘述（prepared statements）與分批擷取（dbSendQuery + dbFetch）能處理塞不進記憶體的結果集。

更大的管線

  • TSDBI 提供時間序列專用的資料庫介面。
  • 存放在 Hadoop／分散式儲存的資料是第二十六章的主題——常見模式是先在叢集上化簡，再把摘要拉進 R。
  • 經驗法則：把吃重的過濾與合併推給資料庫（WHERE、GROUP BY），只把分析真正需要的部分帶進 R。
