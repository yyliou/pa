檢定的邏輯

每個古典檢定都遵循同一套劇本：設定虛無假設 \(H_0\)，計算一個在 \(H_0\) 下分布已知（第十七章）的統計量，回報 p 值——若 \(H_0\) 為真，出現至少這麼極端結果的機率。

Note

怎麼讀 p 值。 p 很小（慣例上 \(< 0.05\)）代表資料在 \(H_0\) 下很令人意外，因此拒絕它。p 很大永遠不能證明 \(H_0\)，只能說資料與它相容。

以下每個檢定都回傳 htest 類別的物件——同樣的結構，同一套閱讀技巧。

單樣本 t 檢定

某型輪胎的失效時間平均值是否異於基準？

temp <- tempfile(fileext = ".rda")
download.file("https://raw.githubusercontent.com/cran/nutshell/master/data/tires.sus.rda", temp, mode = "wb")
load(temp)
spd <- tires.sus$Time_To_Failure[tires.sus$Tire_Type == "E"]
t.test(spd, mu = 8)

    One Sample t-test

data:  spd
t = -30.475, df = 14, p-value = 3.359e-14
alternative hypothesis: true mean is not equal to 8
95 percent confidence interval:
 3.772004 4.327996
sample estimates:
mean of x 
     4.05

常態性與分布檢定

set.seed(3); z <- rnorm(100)
shapiro.test(z)                       # H0：資料為常態

    Shapiro-Wilk normality test

data:  z
W = 0.97928, p-value = 0.1167
ks.test(z, "pnorm")                   # 對照任何具名分布

    Asymptotic one-sample Kolmogorov-Smirnov test

data:  z
D = 0.066739, p-value = 0.7646
alternative hypothesis: two-sided
qqnorm(z); qqline(z)

Q–Q 圖是視覺搭檔：點貼著直線支持常態——還能看出偏離發生在哪裡。

  • 經驗法則：期望次數低於 5 → 改用 fisher.test()。
  • mcnemar.test() 處理成對的類別資料（同一群受試者的前後比較）。

選擇檢定的地圖

  1. 單一數值樣本對照基準：t.test(x, mu=)；非常態 → wilcox.test。
  2. 兩個數值樣本：獨立 → t.test(x, y)；成對 → paired=TRUE；非常態 → wilcox.test。
  3. 三組以上：aov() → TukeyHSD()；非常態 → kruskal.test。
  4. 比例：prop.test / binom.test。
  5. 兩個類別變數：chisq.test / fisher.test。

在相信 p 值之前先檢查假設（常態、等變異、獨立）——並記住統計顯著不等於實務重要。
