歡迎來到 R 的世界

什麼是 R？

R (R Core Team 2024) 是一套自由、開源的統計運算與繪圖環境，源自貝爾實驗室開發的 S 語言 (Becker et al. 1988)。在本課程中，「R」這個名字其實同時指涉好幾件事：

  • 一種以資料為核心設計的程式語言；
  • 執行該語言程式的直譯器（interpreter）；
  • 用來繪製統計圖形的繪圖系統；
  • 一個把直譯器、標準套件與使用者介面打包在一起的 Windows、macOS 或 Linux 桌面應用程式。

本課程教科書為 R in a Nutshell（第二版）(Adler 2012)。第一章的目標非常實際：讓你的電腦裝好一份能用的 R。

為什麼選 R？

Note

R 在應用資料工作中獨佔鰲頭的三個理由。 它完全免授權費、可在所有主流平台上執行，而且擁有龐大的外掛套件生態系，幾乎涵蓋所有發表過的統計方法。

  • R 由跨國的核心開發團隊維護，透過 CRAN（Comprehensive R Archive Network，R 綜合典藏網）發布。
  • 由於原始碼公開，任何人都能檢視方法的實作細節——也能把 R 移植到新平台。這份開放性正是各大系統都有現成安裝檔的原因。
  • 修這門課所需的一切軟體，花費恰好是零元。

R 的版本

發布週期

  • R 核心團隊按固定節奏發布正式版：教科書年代是一年兩次（春、秋）；現在則是每年春季推出一個 x.y.0 主要版本，之後以修補版（x.y.1、x.y.2……）修正錯誤。
  • 版本編號遵循 major.minor.patch 格式，例如 4.4.2。
  • 典型的新版本內容是錯誤修正、效能改善與少量新函式。真正動到語言本身的變更非常罕見，而且通常只涉及冷僻角落（書中舉的例子：R 2.5 改變了未完整初始化陣列中 NA 的型別）。

版本穩定性：為什麼不必擔心

  • 教科書的程式碼是以 R 2.15.1（2012 年）檢驗的。因為語言極其穩定，書中幾乎每個範例至今行為不變；凡是現代 R 有差異之處，本講義一律採用現代行為。
  • 下載連結同理：檔名內嵌版本號（如 R-4.4.2-win.exe），你實際看到的名稱永遠比任何截圖或書本更新。直接抓最新版即可。
  • 針對某個近期版本寫的程式碼，幾乎都能原封不動地在另一版本執行，相鄰版本的結果應該完全一致。

該用哪個版本？

  • 本課程：用 CRAN 上最新的正式版。
  • 隨時可以查看自己執行的版本：
R.version.string
[1] "R version 4.6.0 (2026-04-24)"
  • 修補版號跟同學不同沒關係——本課程做的所有事情，結果都會一模一樣。

取得並安裝 R 執行檔

什麼是 binary？去哪裡找？

Binary（執行檔）指預先編譯好、下載即可執行的 R——這是各桌面平台最簡單的安裝途徑：

  1. 造訪官方網站 https://cran.r-project.org，點選 Download 連結。
  2. 下載連結會帶你到依國家排列的鏡像站（mirror）清單。CRAN 在全球設有副本；地理上近的鏡像站通常網路距離也近、速度快。現代的「0-Cloud」鏡像會自動把你導向合適的伺服器，是安全的預設選擇。
  3. 挑選符合你作業系統的安裝檔並執行。

以下是各平台的注意事項——Windows 與 macOS 使用者通常自行下載安裝檔執行；Linux 使用者則建議交給套件管理系統。

Windows

  • 點 Download R for Windows → base → 執行 .exe 安裝檔。
  • 安裝過程跟其他 Windows 軟體一樣：有足夠權限就很簡單，沒有就麻煩（公司或機房電腦常見）。
    • 最簡單的解法：以一般使用者身分，把 R 裝進自己的檔案空間（例如使用者目錄下）。
    • 或者改用系統管理員帳號安裝——之後安裝套件也得用同一帳號。
  • 預設安裝位置在 %ProgramFiles%\R（通常是 C:\Program Files\R\），並在開始功能表建立 R 群組。
  • 教科書年代 CRAN 只提供 32 位元 Windows 版；現在的安裝檔與 R 本身在現代機器上都是 64 位元，不必做任何選擇。

macOS

  • 點 Download R for macOS，選擇符合硬體的版本：
    • Apple Silicon（M 系列晶片）：arm64 安裝包。
    • Intel Mac：x86_64 安裝包。
  • .pkg 安裝檔會把名為 R 的應用程式放進「應用程式」資料夾，並在命令列路徑加入 R 指令：下載、雙擊、照指示進行即可。
  • 跟所有應用程式一樣需要足夠的權限——個人筆電只要記得自己的密碼；受管理的電腦可能得請管理者協助。
  • 較舊的硬體或系統？CRAN 保留早期 R 版本的舊版安裝檔，可能更適合老系統。（書中年代為 OS X 10.4／10.5 提供不同的 universal binary——同樣的概念，早了二十年。）
  • 若抓錯晶片版本，R 會拒絕啟動——不確定時查一下「關於這台 Mac」。

Linux 與 Unix：套件管理系統

在 Linux 上最乾淨的途徑是發行版的套件管理系統：它會抓取 R、自動解決 R 需要的其他軟體，日後升級也只要一行指令。開始前請確認你知道 root 密碼或擁有 sudo 權限；否則需要系統管理員幫忙。

# Debian / Ubuntu
sudo apt-get update
sudo apt-get install r-base r-base-dev

# Fedora / Red Hat（書中年代的指令是 sudo yum install R.x86_64）
sudo dnf install R
  • 日後更新同樣簡單：sudo apt-get upgrade 或 sudo dnf update R 就能升到最新打包版本。
  • CRAN 自己也維護儲存庫，版本比多數發行版內建的更新；加入方式見 CRAN 的 Linux 說明頁。
  • 順手裝 r-base-dev（或對應套件）很值得——它提供從原始碼編譯外掛套件所需的編譯器。

Linux 與 Unix：手動下載安裝

如果你偏好手動（或沒有套件管理系統可用），也可以自行下載檔案安裝：

  • CRAN 為多個發行版提供預編譯套件——Debian、Ubuntu、Fedora/Red Hat、SUSE——還有 Solaris 版本。
  • Red Hat 系的工具是 RPM。書中範例：把 R-2.15.1.fc10.i386.rpm 下載到 ~/Downloads 後，這樣安裝：
rpm -i ~/Downloads/R-2.15.1.fc10.i386.rpm
# Debian 系的對應指令：
# sudo dpkg -i r-base-core_*.deb
  • 其他 Unix 系統常有自己的管道——例如 FreeBSD 可透過 Ports 系統取得 R。安裝第三方軟體的方法請查閱系統文件。

從原始碼編譯 R

你該自己編譯 R 嗎？

在 Linux／Unix 上從原始碼編譯是標準做法——但在 Windows 或 macOS 上不是：過程棘手，對一般使用者好處有限：

  • 不會省空間——編譯得先下載一堆輔助工具（包括 LaTeX）；
  • 不會省時間——除非工具一應俱全而你的網路又奇慢無比；
  • 唯一實質的好處是榨出額外效能，但實務上 R 的效能即使在很大的資料集上也很少成為瓶頸。

不過，「到處都能編譯」正是 R 得以登上各種冷門平台的原因——開放原始碼的直接好處。

經典編譯流程

給好奇的同學：在類 Unix 系統上編譯，遵循行之有年的儀式：

tar -xzf R-4.4.2.tar.gz
cd R-4.4.2
./configure
make
sudo make install
  • configure 檢查系統並準備編譯設定；make 進行編譯；make install 把成品安裝到位。
  • 教科書把細節留到後面講 R 內部運作的章節；最權威的說明請見 CRAN 上的手冊 R Installation and Administration。

收尾

檢查安裝是否成功

啟動 R，確認主控台有回應：

1 + 1
[1] 2
sessionInfo()$R.version$version.string
[1] "R version 4.6.0 (2026-04-24)"

看到合理的輸出，就表示安裝成功，可以進入第二章認識使用者介面了。

Tip

讓 R 保持最新。 新版本發布時（CRAN 首頁會公告），升級步驟與安裝完全相同——在 Linux 上，套件管理系統還會自動幫你完成。
