為什麼整備很重要

真實資料總是凌亂的：分散在多張表、型別錯誤、重複、滿是缺漏。在任何模型或圖表之前，你都得先合併、轉換、清理、取子集、彙總與重塑。本章就是那個工具箱。貫穿本章的例子是舊金山房屋成交資料：

temp <- tempfile(fileext = ".rda")
download.file("https://raw.githubusercontent.com/cran/nutshell/master/data/sanfrancisco.home.sales.rda", temp, mode = "wb")
load(temp)
dim(sanfrancisco.home.sales)
[1] 3281   15
names(sanfrancisco.home.sales)[1:10]
 [1] "line"       "county"     "street"     "city"       "zip"       
 [6] "date"       "price"      "bedrooms"   "squarefeet" "lotsize"

合併資料集

表格拼接

a <- data.frame(id = 1:3, x = c(10, 20, 30))
b <- data.frame(id = 4:5, x = c(40, 50))
rbind(a, b)                  # 疊加列（欄位需相同）
  id  x
1  1 10
2  2 20
3  3 30
4  4 40
5  5 50
cbind(a, flag = c(TRUE, FALSE, TRUE))   # 增加欄（長度需相同）
  id  x  flag
1  1 10  TRUE
2  2 20 FALSE
3  3 30  TRUE

rbind() 依名稱對齊欄位；面對許多檔案時，do.call(rbind, list_of_frames) 能一次疊完整個串列。

依共同欄位合併

merge() 是 R 的關聯式 join：

sales <- data.frame(id = c(1, 2, 3, 4), price = c(300, 150, 220, 410))
zips  <- data.frame(id = c(1, 2, 3, 5), zip = c("94114", "94110", "94117", "94109"))
merge(sales, zips)                       # 以共同欄位做 inner join
  id price   zip
1  1   300 94114
2  2   150 94110
3  3   220 94117
merge(sales, zips, all.x = TRUE)         # left join：保留所有 sales
  id price   zip
1  1   300 94114
2  2   150 94110
3  3   220 94117
4  4   410  <NA>
  • by=、by.x=、by.y= 控制鍵欄位；all=TRUE 是 full outer join。
  • 合併後務必檢查列數——鍵意外重複會讓列數倍增。

轉換

建立與修改欄位

sales$price.per.k <- sales$price / 1000
sales <- transform(sales, log.price = log(price))
sales
  id price price.per.k log.price
1  1   300        0.30  5.703782
2  2   150        0.15  5.010635
3  3   220        0.22  5.393628
4  4   410        0.41  6.016157

transform() 在資料框內部求值運算式，省去一再重複的 sales$ 前綴。

apply 家族處理資料

nums <- data.frame(a = 1:4, b = c(2, 4, 6, 8))
sapply(nums, mean)             # 每欄一個摘要
  a   b 
2.5 5.0 
mapply(function(x, y) x * y, nums$a, nums$b)
[1]  2  8 18 32
  • lapply() 回傳串列；sapply() 在可行時簡化成向量。
  • 矩陣的逐列運算用 apply(m, 1, f)。

資料分組（binning）

cut() 把連續值變成類別：

price <- sanfrancisco.home.sales$price
bins <- cut(price, breaks = c(0, 5e5, 1e6, 2e6, Inf),
            labels = c("<500K", "500K-1M", "1M-2M", ">2M"))
table(bins)
bins
  <500K 500K-1M   1M-2M     >2M 
    486    1932     697     166

lattice 的 shingle() 製作互相重疊的分組；tabulate() 與 table() 負責計數。

清理資料

找出並移除重複

v <- c("a", "b", "a", "c", "b")
duplicated(v)
[1] FALSE FALSE  TRUE FALSE  TRUE
unique(v)
[1] "a" "b" "c"

對資料框而言，duplicated(df) 標記重複的列——每次合併或從可能重複餵資料的系統匯入後都該跑一次。

排序

sort(c(3, 1, 2))
[1] 1 2 3
o <- order(sales$price, decreasing = TRUE)   # 排序索引
sales[o, ]
  id price price.per.k log.price
4  4   410        0.41  6.016157
1  1   300        0.30  5.703782
3  3   220        0.22  5.393628
2  2   150        0.15  5.010635

Tip

向量用 sort()，資料框用 order()。 order() 回傳排序所需的排列，因此能重排整列——還能用額外鍵打破平手：order(zip, -price)。

缺失值

x <- c(2, NA, 5, 9)
is.na(x)
[1] FALSE  TRUE FALSE FALSE
mean(x); mean(x, na.rm = TRUE)
[1] NA
[1] 5.333333
complete.cases(data.frame(a = c(1, NA), b = c(2, 3)))
[1]  TRUE FALSE
  • na.omit(df) 丟掉不完整的列；記得記錄你丟了幾筆。
  • 要刻意決定：刪除、補值，或為缺漏建模——唯一錯誤的選擇是默不作聲。

子集與抽樣

取子集

expensive <- subset(sales, price > 200, select = c(id, price))
expensive
  id price
1  1   300
3  3   220
4  4   410
# 中括號等價寫法：
sales[sales$price > 200, c("id", "price")]
  id price
1  1   300
3  3   220
4  4   410

subset() 讀起來順；中括號利於程式化。兩者都要會。

隨機抽樣

set.seed(42)                       # 可重現性！
sample(1:10, 4)                    # 不放回抽樣

彙總資料

tapply 與 aggregate

分組摘要一行搞定：

tapply(sanfrancisco.home.sales$price,
       sanfrancisco.home.sales$bedrooms, median, na.rm = TRUE)[1:6]
      0       1       2       3       4       5 
 750000  559000  760000  875000  987500 1020000 
aggregate(price ~ bedrooms, data = sanfrancisco.home.sales,
          FUN = median)[1:6, ]
  bedrooms   price
1        0  750000
2        1  559000
3        2  760000
4        3  875000
5        4  987500
6        5 1020000
  • tapply(values, groups, f) 回傳陣列；aggregate(formula, data, f) 回傳整齊的資料框。
  • by() 是對整個資料框區塊的 tapply。

rowsum 與計數

rowsum(sales$price, group = c("g1", "g1", "g2", "g2"))
   [,1]
g1  450
g2  630
table(sanfrancisco.home.sales$bedrooms)[1:8]

  0   1   2   3   4   5   6   7 
910 216 543 373 176  42  12   6

給 table() 兩個引數就是交叉表——通往第十八章列聯表檢定的大門。

重塑資料

寬與長

同一份資料可以排成寬格式（一個測量一欄）或長格式（一筆測量一列）。建模與 ggplot2 通常偏好長格式。

wide <- data.frame(id = 1:2, q1 = c(90, 70), q2 = c(85, 95))
long <- reshape(wide, direction = "long",
                varying = c("q1", "q2"), v.names = "score",
                timevar = "quiz", times = c("q1", "q2"))
long
     id quiz score
1.q1  1   q1    90
2.q1  2   q1    70
1.q2  1   q2    85
2.q2  2   q2    95

stack 與現代工具

stack(wide, select = c(q1, q2))
  values ind
1     90  q1
2     70  q1
3     85  q2
4     95  q2
  • Base R：簡單情況用 stack()/unstack()，一般情況用 reshape()。
  • 現代實務上，tidyr 套件（pivot_longer()、pivot_wider()）以更易讀的方式表達同樣的操作；進入 tidyverse 世界時就會遇到它。
